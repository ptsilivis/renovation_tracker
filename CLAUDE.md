# CLAUDE.md — Kampos Hub

Shared dashboard for a family summer-house renovation (stone house in Messinian
Mani, ~€50k). Four family members, all admin. Finances + activities in one place.
Full requirements: `doc/prds/kampos-hub.prd.md`. Design/brand: `PRODUCT.md`,
`DESIGN.md`. Origin: a Claude Design project turned into a real multi-user app.

## Stack

- **Backend** `backend/` — Python 3 + FastAPI + SQLAlchemy 2 + Alembic, PostgreSQL,
  argon2 + JWT (httpOnly cookie). Serves the API and the static frontend.
- **Frontend** `frontend/` — vanilla JS/HTML/CSS, **no build step**, ES modules.
  A `fetch()` client (`js/api.js`) is the single data layer.
- **Host** — Raspberry Pi + Cloudflare Tunnel. See `deploy/DEPLOY.md`.

## Layout

```
backend/app/
  main.py         FastAPI app; routers mounted /api/*, static frontend at /
  models.py       ORM: users, settings, categories, tasks, cost_items, rooms,
                  surfaces, moodboard_items, phases, plan_rooms/walls/underlays, activity
  crud.py         generic collection registry + serialize/create/update/remove/bulk_create
  routers/        auth, collections (CRUD + /data snapshot + /{c}/bulk), meta, files
  auth.py deps.py config.py db.py seed.py   (seed = Greek sample data + 4 accounts)
  alembic/        migrations (autogenerated)
frontend/js/
  api.js state.js i18n.js router.js ui.js compute.js glb.js gantt.js app.js
  screens/{overview,tasks,costs,moodboard,floorplan,measurements}.js
```

## Data flow / conventions

- `GET /api/data` returns one snapshot: `{ settings, activity, <collection>: [...] }`.
  Collection API names are the snake_case table names (`cost_items`, `plan_walls`, …).
- Frontend mutation path (`state.js`): `create/update/remove/patchSettings` call the
  API, then re-pull the whole snapshot and re-render. **Last-write-wins**, refresh to
  sync. Fine for 4 users; do not add optimistic caching without reason.
- `plan_rooms/walls/underlays` store free-form geometry in a JSONB `data` column;
  the API flattens them to `{id, ...data}`.
- Money/measurements: `.tnum` (tabular numerals), `money()` formats €, variance
  colored (over = terracotta `--accent`, under = green `--ok`).
- Bilingual EL/EN: `i18n.js` `t(key)`; `localName(obj)` picks `name_el`/`name_en`.
  Data strings stay Greek; only chrome translates.
- Roles: everyone is `admin`; the delete gate is still enforced server-side.

## Notable features

- **Overview Gantt** (`js/gantt.js`): editable phase names, drag bars / edges for
  dates (month snap), click a row for a milestone diamond, drag ⠿ to reorder,
  `settings.project_start/end` frame the axis. Budget stat card is click-to-edit.
- **Floor plan** (`screens/floorplan.js`): SVG rooms/walls, drag + resize, image
  underlay. **GLB import** (`js/glb.js`) parses the model, keeps vertical
  surfaces, projects top-down, **fits uniformly to canvas** (keeps layout faithful),
  merges collinear fragments, and creates editable `plan_walls` via `/bulk`.
  `settings.plan_scale` (px per metre) comes from the fit; dimensions show metres;
  selecting a wall/room lets you type exact measurements. Do NOT reintroduce a
  boundary-edge filter or per-model real-scale centering — both scrambled the layout.

## Running locally

```bash
# Postgres: docker compose up -d db  (or any local Postgres; edit backend/.env)
cd backend && python3 -m venv .venv && ./.venv/bin/pip install -r requirements.txt
cp .env.example .env
./.venv/bin/alembic upgrade head
./.venv/bin/python -m app.seed          # sample data + 4 accounts (pw: kampos2026)
./.venv/bin/uvicorn app.main:app --reload --port 8000
# open http://localhost:8000  → login p.tsilivis10@gmail.com / kampos2026
```

Schema change → edit `models.py`, `alembic revision --autogenerate -m "..."`,
`alembic upgrade head`. Snapshot/patch for `settings` live in `routers/meta.py` and
`routers/collections.py` — update both when adding a settings field.

## Verifying without a browser

No browser/Playwright in the dev sandbox. Backend is tested with curl. The
frontend is smoke-tested by rendering the real screen modules under **jsdom**
against the live API (see the harness pattern used in scratchpad): render each
screen in EL + EN, assert no runtime error. **Cannot** verify here: pointer drag
feel, and three.js GLB parsing (loads from CDN via the import map in `index.html`).
Flag these for the user to check live.

## Gotchas

- Dev DB in this environment ran on a throwaway Postgres at **:5433**; `backend/.env`
  may point there. README/DEPLOY assume docker `:5432`. `.env` is gitignored.
- `POST /api/{collection}/bulk` and explicit routes (`/api/files`, `/api/activity`)
  must be registered before the generic `/api/{collection}` catch-all in `main.py`.
